# ☁くもぼうやミニゲームの開発リポジトリ☁  


## 🔰 制作の背景と目的  
本作は、Unity・C#の学習成果物として制作しました。  
制作に至った背景は、単にコードの書き方を学ぶだけでなく、ゲームとしての面白さを技術でどう表現するかをテーマに学習を進めたかった為です。  
学んだ基礎知識のアウトプットを目的としており、ゲーム案の設計やUnityによる開発、GitHubでの公開までを独学で完遂しました。  
初めて開発に挑戦し約2ヶ月間試行錯誤を重ねた結果、自身の現時点での全力を出し切った完成度の高い作品になったと自負しております。  


## 📚 学習における注力ポイント
- 試行錯誤したギミック: 物理演算やお邪魔バーのランダム生成などを活用し、ランダム性の高いゲーム性を実現。  
- データの永続化: スコアやフィードバックなどの要素をサーバーやDBを介して保存することで、外部システムとの連携や基礎的な仕組みを網羅。  
- 日本語入力環境の改善: WebGLの標準仕様では日本語を入力することが困難であり、これを可能にするため外部ライブラリを導入。  
Webフロントエンドの最適化: WebGLビルド後の`Index.html`をカスタマイズし、ブラウザ上での表示崩れの防止や、Webページとしてのレイアウト調整を実施。

- ArcadeScoreManager.cs  
    Unity Gaming Services（UGS）のLeaderboardsを利用した、通常モード用のスコアランキングシステム。1人につき最高得点のみを保存する仕組みではなく、アーケードゲームのように同じプレイヤーが何度でも登録できる方式を採用した。  
    サーバーとの通信を待つ間にゲームそのものがフリーズしないよう、Unity Servicesの初期化・ログイン処理・ランキングへの送信は、非同期処理にて実行する仕組みとなっている。  
    `OnClickSubmit`関数に`await`から始まる行があり、スコアの送信が終わる前に次のシーンに移行しスコアが登録されないという悲しい事件が起きないようになっている。  

- BarGenerator.cs  
    ゲーム開始時、降ってくる文字の軌道を変える障害物（回転バーや回転六角形）をランダムに生成するスクリプト。プレイする度に配置が変化するため、単一の攻略パターンに囚われない、動的なゲームプレイを実現している。  
    また、配置だけでなく、障害物の出現数（10〜15個）や、回転速度・方向の変化、形状の抽選（棒は必ず出るが六角形は出ないこともある）などにより、多角的なランダム要素が絡み合う設計となっている。  

- ColorSelector.cs  
    降ってくる文字やUIのテキストカラーを、ユーザーが選択した任意の色へ変更可能にするスクリプト。通常モードのランキング登録時にも、この設定がプレイヤー名の文字色へと反映される。  
    シングルトンパターンを活用し、`GameManager.instance`を介してシーン間でのデータ引き継ぎを実現している。  

- CountResetWall.cs  
    ゲームプレイ中、特定の文字がゴールの壁に接触する度に、デバフ発動までのカウントをリセットするスクリプト。  
    2種類あるデバフを同時にリセットするのではなく、交互にリセットする仕様にすることでゲームのテンポを制御している。プログラム内部では、`bool`フラグを用いたトグル処理により、単一のトリガーから交互にリセットする機能を実現した。  
    また、"特定の文字"の識別にはタグによるフィルタリングを活用し、対象を明確に区別することで意図しない挙動を防ぐ設計としている。  

- FirebaseManager.cs  
    WebGLビルドにおいて、ブラウザ側のJavaScriptと連携してFirebaseへデータを送信する外部連携スクリプト。  
    `DllImport("__Internal")`を用いたプラグイン連携により、Unityの枠組みを超えてJavaScript関数を呼び出し、Firebaseを利用する仕組みを構築した。  
    また、環境ごとのエラーを未然に防ぐためプリプロセッサ（条件付きコンパイル）やif文を活用し、Unityエディタ上ではログ出力のみ、Web公開環境では実際の通信を行うよう挙動を制御する設計となっている。  

- FreeModeGuide.cs  
    フリーモード開始時に特殊操作の説明を表示するスクリプト。  
    表示から５秒後にフェードアウトすることでユーザビリティと没入感を両立させている。  

- FreeModeNanyateManager.cs  
    フリーモードのスコア計算・デバフ判定・演出制御を一括管理するマネージャースクリプト。  
    通常モードの設計をベースに、フリーモード独自のゲームバランスへと調整している。  
    専用演出として100ポイント消費と引き換えに発動する「くもぼうやのポップコーンエフェクト」は、高負荷な`Rigidbody2D`を使わず、コルーチン`IEnumerator`を用いた自前計算により低負荷な設計を実現した。また、演出終了時には`Destroy`で確実にオブジェクトを消去しており、パフォーマンス低下を防いでいる。  

- GameManager.cs  
    ゲーム全体のライフサイクル（開始・進行・終了）と状態管理を中央集権的に統括する、ゲームシステムの核となるマネージャースクリプト。  
    シーンロードイベント`sceneLoaded`により、読み込まれたシーン名から「通常モード」と「フリーモード」を自動で判別し、モードに応じた初期化やUIの出し分け（制限時間の有無など）を実行する仕組みになっている。  
    また、ゲーム全体のライフサイクルの遷移を、1本のコルーチン`GameLoopCoroutine`によるステート管理で制御している。  

- GoalSensor.cs  
    ゲームプレイ中、得点の判定基準となる文字がゴールを通過したかを判定するスクリプト。  
    `GetComponentInChildren`関数を用いて親オブジェクトへの接触から子要素のテキスト情報を抽出する構造としており、一度ゴールした文字のタグを変更することで、同一フレームや連続接触による「スコアの重複加算」を未然に防ぐ設計となっている。  

- GravityController.cs  
    ゲームプレイ中、2種類あるデバフの1つである「重力」を制御するスクリプト。  
    `Physics2D.gravity`を直接操作することで、個別オブジェクトにコードを記述することなく、シーン全体の物理オブジェクトを一括制御する構造としている。また、変数を `{ get; private set; }`（プロパティ）にすることで、外部からの状態参照を許可しつつ、意図しない書き換えを防ぐ安全な設計（カプセル化）を実現している。  

- HowToManager.cs  
    タイトル画面から開くことができる、遊び方の画面を表示するスクリプト。  

- InformationManager.cs  
    タイトル画面から開くことができる、お問い合わせ・フィードバック送信の画面を表示するスクリプト。フロントエンド処理と、スパム防止のセキュリティロジックを制御している。  
    Firebase等のバックエンド保護、およびサーバー負荷軽減のため、`PlayerPrefs`を用いた送信制限システム（1時間に5回まで）を実装している。前回の送信時刻（System.DateTime）からの経過時間をローカルに保存・判定することで、悪質な連続送信（スパム行為）を未然にブロックする仕組みとなっている。  

- InputField.cs  
    スコアをランキング登録する際のユーザー名入力をひらがなのみに限定するスクリプト。正規表現（Regex）を活用した仕組みとなっている。  

- InvisibleGoall.cs  
    ゲームプレイ中、2種類あるデバフの1つである「ゴール隠蔽」を制御するスクリプト。  
    オブジェクト自体をSetActive(false)にするのではなく、enabledだけを操作することで、当たり判定は残したまま見た目だけを消す仕組みとなっている。  
    ※スクリプト名が Goall とタイポしていますが、リファクタリング時の不具合防止として（あと初開発の思い出のため）、あえてそのまま残しています。

- KumoGyurururuSpin.cs  
    ランキング画面での隠し演出を制御するスクリプト。  
    画像を回転させる処理をしており、`Update`ではなく`FixedUpdate`を使用することで、物理演算（`Rigidbody`等）と同期した滑らかな回転を目指した。また、停止時に単に止めるだけでなく、`Quaternion.identity`により角度をリセットする仕様とした。  

- MobileInputManager.cs  
    モバイル端末（スマートフォンやタブレットを想定）での操作に対応するため、画面上に移動およびシュート用のバーチャルUIを表示・制御するスクリプト。  
    所持しているiPhone 15環境でのみ正常動作を検証・確認済み。その他のモバイル端末における互換性テストは、検証環境がないため今後のサポートや最適化を行う予定はなく、本スクリプトの開発・保守は現状をもって終了とする。  

- MojiController.cs  
    ゲームプレイ中、落下する文字オブジェクトの生成時の初期化・種類（ひらがな50音の中から）の抽選・落下処理・画面外まで移動した際に破棄するサイクルを制御するスクリプト。  
    文字生成時、プレイヤーの狙うべき正解文字「な・ん・や・て」が50%の確率で確実に現れるよう補正ルートを実装。残りの50%の"全文字からの完全ランダム抽選に回った"際にも、偶然正解文字が選ばれた場合にタグが正しく Target に切り替わるよう、ループによる二重チェックを施し、ゲーム判定の不整合を防いでいる。  

- MojiSpawner.cs  
    ゲームプレイ中、判定対象となる文字オブジェクト（プレハブ）を上空から定期的・ランダムに生成し続けるスポナーを制御するスクリプト。  
    `InvokeRepeating`を用いた軽量な定期実行システムとなっており、毎フレームの監視が必要な`Update`内でのタイマー計算を避け、Unity標準の機能を活用。CPU負荷を最小限に抑えつつ、設定された間隔ごとに安定してオブジェクトを生成する構造になっている。  

- MovingGoal.cs  
    ゲームプレイ中、バスケットゴールの見た目をしたゴールが上下に移動するスクリプト。  
    ゴールを動的なギミックにすることで、単調なスコア獲得を防ぎ、プレイヤーのエイムやタイミングの駆け引きといったゲームの難易度と戦略性を引き上げる役割を担っている。  

- PauseButton.cs  
    ゲームプレイ中に表示される一時停止ボタンについて、`ColorSelector.cs`にて選択した色を反映させるためのスクリプト。  

- PauseManager.cs  
    ゲームプレイ中、一時停止画面を表示・制御するためのスクリプト。`PauseButton.cs`が当てられたボタンを押下することにより表示される。  
    ポーズ画面からタイトルへ戻り、ゲームを再開する行為を繰り返すと、前シーンの重力値が累積され文字の落下速度が異常高速化するバグを解消する為、`Start`関数内で物理エンジンの重力設定`（Physics2D.gravity）`を完全に初期化する仕組みとした。  

- PlayerController.cs  
    ゲームプレイ中、プレイヤーキャラクター「くもぼうや」の移動制限・文字オブジェクトのキャッチ＆ホールド・文字投げシステムを制御するスクリプト。  
    文字をホールドする際は、オブジェクトのレイヤー変更と`Rigidbody2D`の`Kinematic`化によって物理演算を一時停止させ、移動中も文字を手に固定保持できる仕様を実現。  
    また、一時停止中に文字投げのパワーを溜め続けるといった不正行為を防ぐために強制的なチャージリセット処理を施すなど、例外処理も実装している。  

- PowerGauge.cs  
    ゲームプレイ中、プレイヤーが文字を投げる際の「チャージパワー（溜め具合）」をリアルタイムに可視化し、プレイヤーへ視覚的なフィードバックを返すUI制御スクリプト。  

- RankingManager.cs  
    Unity Gaming Services（UGS）のリーダーボード機能と連携し、ランキングデータの非同期取得およびランキング画面表示を制御するスクリプト。  
    シーン開始時に`async/await`（非同期処理）を用いてUnityサービスを初期化し、`SignInAnonymouslyAsync`による匿名サインインを確立。  

- RankingScrollView.cs  
    ランキング画面にて、ランクイン外のプレイヤーを強制スクロールで表示する画面のスクリプト。  
    スクロールは`LateUpdate`にて座標更新を行うことで実装している。また、スクロール中に`ScrollRect`本来の物理的な慣性速度`（velocity）`が干渉して挙動が不安定になるのを防ぐため、毎フレーム速度をほぼゼロ`（Vector2(0, 0.0001f)）`に固定化している。  

- ReadmeHandler.cs  
    本作品のREADMEを開く画面を表示するスクリプト。  

- ResultManager.cs  
    通常モードにてプレイし、時間制限が来た際のTimeUpの表示・リザルト画面の表示・スコアの集計を実施するスクリプト。    

- SceneChanger.cs  
    シーン切り替えを管理するスクリプト。    

- ScoreManager.cs  
    ゲームプレイ中のスコア加減点システム・当たり文字（な・ん・や・て）の獲得数カウント・デバフ状態に応じた加点ロジックを管理するスクリプト。  

- SecretLoader.cs  
    WebGLプラットフォームにおけるUnityとブラウザ（JavaScript）間の相互連携、および外部設定ファイルを用いたFirebase等の初期化検証を担うバックエンド連携スクリプト。  
    `DllImport("__Internal")`を用いて、ブラウザ（JavaScript）側で定義されたグローバル関数`（setupFirebase）`をUnity C#側から直接呼び出す外部連携インターフェースを構築。  
    サーバーとの通信を待つ間にゲームそのものがフリーズしないよう、非同期処理にて実行する仕組みとなっている。  

- TitleManager.cs   
    タイトル画面でのボタンを制御するスクリプト。  

- YokobouRotateObstacle.cs  
    降ってくる文字の軌道を変える障害物（回転する棒）を制御するスクリプト。

## 🗝 APIキーに関する注記  
本プロジェクトの`index.html`にはFirebaseのAPIキーが含まれています。これは Firebase JS SDK の仕様に基づく公開前提の識別子です。  
データの保護については、Firebase Security Rulesを用いてサーバーサイドで厳格に制限（書き込みのみ許可・読み取り禁止等）を行っており、セキュリティ上の問題がないことを確認済みです。  
また、GitHub からシークレット検出の警告が出る場合がありますが、意図的な構成です。
